# FLOWTEXT Git 提交规范

## 分支策略

| 分支 | 用途 |
|------|------|
| `main` | 稳定版本，可发布 |
| `dev` | 日常开发主分支 |
| `feature/*` | 功能开发分支（完成后合并到 dev） |
| `fix/*` | 修复分支 |

开发过程中，直接使用 `main` 分支进行提交（单⼈开发简化流程），每次提交对应一个可运行的小步骤。

## Commit Message 格式

```
<type>(<scope>): <简短描述>

[可选的详细说明]
```

### Type 类型

| Type | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | 修复 bug |
| `docs` | 文档变更 |
| `style` | UI/样式变更（非功能） |
| `refactor` | 重构 |
| `perf` | 性能优化 |
| `chore` | 构建/配置变更 |
| `init` | 项目初始化 |

### 示例

```
init: 初始化项目结构和文档
feat(editor): 实现 Markdown 实时渲染
feat(toolbar): 添加浮动格式工具栏
style(ui): 优化 Apple 风格配色和间距
docs: 更新开发步骤文档
fix(editor): 修复光标跳转问题
```

## 提交频率

- 每个小功能点完成一次提交
- 提交前确保应用能正常启动
- 提交信息用中文描述

## 流程

```bash
git add <文件>
git commit -m "feat(scope): 描述"
git push origin main
```
