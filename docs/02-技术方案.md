# FLOWTEXT 技术方案

## 整体架构

```
┌─────────────────────────────────────────┐
│            Electron 主进程               │
│  (main.js) 窗口管理、文件 I/O、菜单栏    │
├─────────────────────────────────────────┤
│          Electron 渲染进程               │
│  (index.html) Markdown 编辑器 UI         │
│  (renderer.js) 编辑器逻辑、工具栏逻辑     │
│  (preload.js) 安全的 IPC 桥接            │
└─────────────────────────────────────────┘
```

## 技术栈明细

| 技术 | 用途 | 版本 |
|------|------|------|
| Electron | 桌面应用框架 | 最新稳定版 |
| marked | Markdown → HTML 解析 | 最新版 |
| highlight.js | 代码块语法高亮 | 最新版 |
| CSS3 | Apple 风格 UI 实现 | — |

## 编辑器方案

### 实时渲染策略
采用 **contenteditable div** 方式实现 Typora 风格实时编辑器：

1. 编辑器区域为一个 `div[contenteditable=true]`
2. 用户输入时，实时解析 innerHTML 中的 Markdown 语法
3. 将解析结果渲染为格式化文本（HTML）
4. 光标位置通过选区 API 精确维护

### 浮动工具栏方案
1. 监听 `selectionchange` 事件检测文字选中状态
2. 选中文字后，在选区上方弹出浮动工具栏
3. 工具栏按钮通过 `document.execCommand` 或直接操作选区 Range 实现格式修改
4. 取消选中时自动隐藏工具栏

## 图片处理方案
1. 拖拽图片：监听 `drop` 事件，读取 File 对象，复制到本地 assets 目录
2. 粘贴图片：监听 `paste` 事件，从剪贴板读取图片数据
3. 文件对话框：通过 Electron dialog API 打开文件选择器

## 导出方案
| 格式 | 方案 |
|------|------|
| HTML | 直接导出编辑器渲染后的 HTML 内容 + CSS |
| PDF | 使用 Electron 的 `webContents.printToPDF()` API |

## 文件操作方案
- 新建：清空编辑器内容，重置文件路径
- 打开：通过 dialog.showOpenDialog 选择 .md 文件，读取内容填入编辑器
- 保存：通过 dialog.showSaveDialog 或直接写入当前文件路径
- 文件编码：统一 UTF-8

## 安全策略
- 启用 Electron 的 contextIsolation
- 使用 preload.js 暴露安全的 IPC API
- 禁用渲染进程的 nodeIntegration
- 对 HTML 输出做 XSS 防护
