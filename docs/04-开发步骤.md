## Phase 2: Markdown 编辑器核心（已完成 ✅）

**目标**：编辑器支持 Markdown 实时渲染。

**步骤：**
1. 安装 marked 和 highlight.js 依赖
2. 在 index.html 中创建编辑器区域（`div[contenteditable=true]`）
3. 实现 Markdown 文本输入 → 实时解析 → 渲染为格式化 HTML
4. 实现光标位置维护（编辑后不跳转）
5. 支持标题（H1-H6）、粗体、斜体、删除线的实时渲染
6. 支持代码块、引用、列表的实时渲染
7. 支持表格的实时渲染
8. 支持链接的实时渲染
9. 支持分割线的实时渲染
10. 代码块语法高亮（highlight.js）

**验证标准：** 输入任意 Markdown 语法，编辑器即时渲染为格式化效果。

---

## Phase 3: 浮动格式工具栏

**目标**：选中文字后弹出浮动工具栏，可进行可视化格式修改。

**步骤：**
1. 创建浮动工具栏 HTML 结构
2. 使用 CSS 实现 Apple 风格工具栏样式
3. 监听 `selectionchange` 事件，检测文字选中
4. 选中文字时计算选区位置，定位工具栏
5. 取消选中时隐藏工具栏
6. 实现粗体按钮
7. 实现斜体按钮
8. 实现下划线按钮
9. 实现删除线按钮
10. 实现字体颜色选择器
11. 实现背景高亮色选择器
12. 实现字体大小选择器
13. 实现标题级别切换（H1-H6）

**验证标准：** 选中文字弹出工具栏，点击各按钮生效。

---

## Phase 4: 图片支持

**目标**：支持拖拽、粘贴、对话框三种方式插入图片。

**步骤：**
1. 实现拖拽图片到编辑器（drop 事件）
2. 实现从剪贴板粘贴图片（paste 事件）
3. 实现通过文件对话框选择图片
4. 图片在编辑器内渲染显示
5. 图片文件存储管理（assets 目录）

**验证标准：** 三种方式均能成功插入并显示图片。

---

## Phase 5: 文件操作

**目标**：支持新建、打开、保存 .md 文件。

**步骤：**
1. 实现新建文件功能
2. 实现打开 .md 文件（Electron dialog API）
3. 实现保存文件（当前路径）
4. 实现另存为文件
5. 文件修改状态提示（未保存标记）

**验证标准：** 能打开外部 .md 文件、编辑后保存、新建文件正常。

---

## Phase 6: 导出功能

**目标**：支持导出为 PDF 和 HTML。

**步骤：**
1. 实现导出 HTML（完整 HTML 文件 + 样式）
2. 实现导出 PDF（Electron printToPDF）
3. 导出版本包含完整样式

**验证标准：** 导出文件格式正确，样式完整。

---

## Phase 7: 界面打磨

**目标**：全面美化 UI，达到 Apple 风格标准。

**步骤：**
1. 自定义标题栏（macOS 风格窗口控制按钮）
2. 优化整体配色和间距
3. 添加过渡动效
4. 优化字体和排版
5. 暗色模式（可选）

**验证标准：** 界面符合设计规范文档中的 Apple 风格标准。
